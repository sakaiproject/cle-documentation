Sakai user documentation is being handled by a sub-set of the Sakai community. Beyond some technical difficulties related to storing current help text in IU's Knowledge Base, it is likely that help content will be managed quite well. What Sakai is missing is a comprehensive set of technical documentation. This information is currently scattered widely throughout Confluence, most of it seriously out of date. This is where the recent TCC documentation effort can really make a difference.
